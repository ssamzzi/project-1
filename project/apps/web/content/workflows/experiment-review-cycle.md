# Experiment Review and Troubleshooting Cycle

This workflow turns calculations, controls, observations, and deviations into a repeatable review process. It is useful after failed experiments and after successful experiments that need to be repeated.

## 1. Collect the setup record

Gather final reagent volumes, stock concentrations, sample IDs, control layout, incubation times, instrument settings, and any deviations from the planned protocol.

## 2. Separate observation from interpretation

Observation describes what was seen. Interpretation explains what it might mean. Keeping them separate reduces overconfidence and makes troubleshooting more precise.

## 3. Sort likely causes

Group possible causes into sample, reagent, instrument, and operator categories. This prevents the review from focusing only on the most emotionally obvious mistake.

## 4. Choose one change

The next run should test a specific idea. Changing many variables at once may produce a better result without explaining why.

## 5. Preserve the learning

Write the next-step decision into the notebook or shared record. The value of troubleshooting is not only solving one failed run; it is making the next similar run easier.

## Review checklist

- Inputs and units checked.
- Controls reviewed.
- Deviations recorded.
- Likely causes grouped.
- Next test defined.
